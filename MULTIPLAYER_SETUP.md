# Realtime Multiplayer Setup

This adds real-time 2-player rooms using Supabase Realtime.

## 1. Add a browser-safe Supabase key in Vercel

Add one of these environment variables:

- `SUPABASE_PUBLISHABLE_KEY`
- or `SUPABASE_ANON_KEY`

Keep using your existing:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

The browser uses the publishable/anon key through `/api/runtime-config`.

## 2. Create the multiplayer table

Run this SQL in the Supabase SQL editor:

```sql
create table if not exists public.game_rooms (
  id bigint generated by default as identity primary key,
  code text not null unique,
  host_player_id text not null,
  host_name text not null,
  status text not null default 'waiting',
  seed text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists game_rooms_code_idx on public.game_rooms (code);
```

## 3. Add open policies for the room table

For this prototype, use simple public access so phones can create/join rooms without auth:

```sql
alter table public.game_rooms enable row level security;

create policy "public can read rooms"
on public.game_rooms
for select
to anon, authenticated
using (true);

create policy "public can create rooms"
on public.game_rooms
for insert
to anon, authenticated
with check (true);

create policy "public can update rooms"
on public.game_rooms
for update
to anon, authenticated
using (true)
with check (true);
```

## 4. Realtime

Supabase Realtime Presence and Broadcast are used for:

- players appearing in lobby
- host start signal
- live race progress updates

No extra database table is required for lobby presence.

## 5. Redeploy

After adding the key and SQL, redeploy on Vercel.
